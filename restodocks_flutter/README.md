# Restodocks Flutter

Полностью переписанное на Flutter приложение для управления рестораном, работающее на iOS, Android и Web.

**Грандплан продукта** (закрыто / в бэклоге): [docs/GRAND_PLAN.md](../docs/GRAND_PLAN.md).

## 🚀 Возможности

### Аутентификация и управление пользователями
- ✅ Регистрация компаний и сотрудников
- ✅ Многоуровневая система ролей (владелец, шеф-повар, менеджеры, персонал)
- ✅ Управление отделами (кухня, бар, зал, управление)
- ✅ Безопасная аутентификация с PIN-кодами

### Управление продуктами
- ✅ Каталог продуктов с разделением по отделам
- ✅ КБЖУ и аллергены для каждого продукта
- ✅ Многоязычные названия
- ✅ Система ценообразования

### Технологические карты (ТТК)
- ✅ Создание рецептов с ингредиентами
- ✅ Расчет себестоимости блюд
- ✅ Автоматический расчет КБЖУ
- ✅ Учет технологических процессов приготовления

### Кроссплатформенность
- ✅ iOS (iPhone, iPad)
- ✅ Android
- ✅ Web (Safari, Chrome, Firefox)
- ✅ Responsive дизайн

## 🚀 Запуск приложения

### Предварительные требования
1. Установлен Flutter SDK
2. Настроен PATH для Flutter
3. Для iOS: установлен Xcode
4. Для Android: установлен Android Studio

### Запуск в браузере (Web)

**По умолчанию — Safari.** Flutter не умеет `-d safari`, поэтому используем web-server и открываем приложение в Safari:

```bash
./run_web_safari.sh
```

Либо вручную:
```bash
flutter run -d web-server --web-port=8080
```
После старта открой в **Safari**: `http://localhost:8080`.

Для Chrome: `flutter run -d chrome`.

### Запуск на iOS симуляторе
```bash
flutter run -d ios
```

### Запуск на Android эмуляторе
```bash
flutter run -d android
```

## 🔐 Система ролей

| Роль | Права |
|------|-------|
| Владелец | Полный доступ ко всем функциям |
| Шеф-повар | Управление кухней, ТТК, расписанием |

| Менеджер | Управление отделом, отчеты |
| Персонал | Доступ к функциям своего отдела |
