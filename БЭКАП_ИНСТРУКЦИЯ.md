# 🚀 ПРОСТОЙ БЭКАП RESTODOCKS

## ОДНИМ НАЖАТИЕМ КЛАВИШИ!

### ВАРИАНТ 1: ДВОЙНОЙ КЛИК НА ФАЙЛЕ
1. **Найдите файл**: `Backup_Restodocks.command`
2. **Двойной клик** на нем
3. **Ждете** пока выполнится бэкап
4. **Готово!** Архив с бэкапом появится в той же папке

### ВАРИАНТ 2: Через терминал
```bash
./backup_all.sh
```

### ВАРИАНТ 3: Создать ярлык на рабочем столе
1. Правой кнопкой на `Backup_Restodocks.command`
2. Выбрать "Создать ярлык" или "Make Alias"
3. Перетащить ярлык на рабочий стол

## 🗄️ ДОБАВЛЕНИЕ БЭКАПА БАЗЫ ДАННЫХ

### ЕСЛИ ПАРОЛЬ УЖЕ ИЗВЕСТЕН:
```bash
./setup_db_password.sh
```
Скрипт попросит ввести пароль и автоматически настроит все!

### ЕСЛИ ПАРОЛЬ НУЖНО СБРОСИТЬ (текущий пароль скрыт):
1. В Supabase Dashboard: Database → Settings → "Reset database password"
2. **СКОПИРУЙТЕ НОВЫЙ ПАРОЛЬ НЕМЕДЛЕННО!**
3. Подождите 1-2 минуты
4. Запустите:
```bash
./after_password_reset.sh
```
5. Введите новый пароль

**✅ БЕЗОПАСНО:** Сброс пароля БД НЕ сломает работу сайта и приложения!

### ТЕСТИРОВАНИЕ НАСТРОЙКИ:
```bash
./check_supabase_setup.sh
```

### РУЧНАЯ НАСТРОЙКА:
1. Получите пароль из Supabase Dashboard
2. Откройте `backup_config.env`
3. Замените `YOUR_DB_PASSWORD_HERE` на настоящий пароль

## 🔧 ПРОБЛЕМЫ?

### Если "что-то в supabase не получается":
1. Запустите `./test_db_connection.sh` - проверит подключение
2. Если пароль не настроен - запустите `./setup_db_password.sh`
3. Проверьте пароль в Supabase Dashboard еще раз

### Если pg_dump не найден:
```bash
python3 -m pip install --user psycopg2-binary
```
