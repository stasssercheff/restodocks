# Настройка бэкапа — один раз (5 минут)

После этой настройки бэкап будет **полным** (код + БД + storage + всё). Делается **только один раз**.

---

## Шаг 1. Postgres.app (для бэкапа и восстановления БД)

### Что это
Утилита `pg_dump` и `psql` — без неё база данных не попадёт в бэкап.

### Как установить

1. Открой в браузере: **https://postgresapp.com**
2. Нажми **Download** (обычно Postgres.app 2.x)
3. Открой скачанный файл → появится окно с иконкой слонёнка
4. **Перетащи** Postgres в папку **Applications** (Программы)
5. Открой **Applications** → запусти **Postgres**
6. В окне Postgres нажми кнопку **Initialize**
7. Готово — можешь закрыть окно Postgres (он работает в фоне)

### Проверка
Открой **Терминал** и введи:
```bash
pg_dump --version
```
Должно вывести что-то вроде `pg_dump (PostgreSQL) 16.x`. Если команда не найдена — **перезапусти Терминал** или перезагрузи Mac.

## Шаг 2. Файл backup_config.env (пароль БД)

### Что это
В этом файле хранится строка подключения к базе данных Supabase. Нужна для бэкапа и восстановления.

### Как создать

1. Открой в браузере:
   ```
   https://supabase.com/dashboard/project/osglfptwbuqqmqunttha/settings/database
   ```
2. В разделе **Database password** нажми **Reveal** или **Reset** и скопируй пароль
3. Открой папку проекта Restodocks в Finder
4. Создай новый файл с именем: `backup_config.env` (точно так)
5. Открой его в текстовом редакторе и вставь **одну строку** (замени `ТВОЙ_ПАРОЛЬ` на скопированный пароль):
   ```
   SUPABASE_DB_URL=postgresql://postgres:ТВОЙ_ПАРОЛЬ@db.osglfptwbuqqmqunttha.supabase.co:5432/postgres
   ```
6. Сохрани файл

### Пример
Если пароль `abc123xyz`, строка будет:
```
SUPABASE_DB_URL=postgresql://postgres:abc123xyz@db.osglfptwbuqqmqunttha.supabase.co:5432/postgres
```

### Важно
- Файл `backup_config.env` **не** коммитится в Git (содержит пароль)
- Храни его только локально
- При восстановлении на другом компьютере — пароль можно снова взять из Supabase Dashboard

## Готово

Теперь запускай бэкап:

- **Двойной клик** по `Backup_Restodocks.command`
- Или в Терминале: `./backup_all.sh`

В конце должно быть:
```
✅ База данных PostgreSQL ✓
```

Если видишь `⚠️ БАЗА ДАННЫХ ОТСУТСТВУЕТ` — проверь, что:
1. Postgres.app установлен и инициализирован
2. В `backup_config.env` правильный пароль (без пробелов, в одну строку)
3. Терминал перезапущен после установки Postgres.app

## Восстановление (когда понадобится)

### Вариант А — в текущем проекте
```bash
./restore_all.sh
# или: ./restore_all.sh backup_20260225_183212_COMPLETE.tar.gz
```

### Вариант Б — на другом компьютере (архив на флешке/облаке)
1. Установи Postgres.app (см. Шаг 1 выше)
2. Распакуй архив: `tar -xzf backup_xxx_COMPLETE.tar.gz`
3. Перейди в папку: `cd backup_xxx`
4. Запусти: `./ВОССТАНОВИТЬ.sh`
5. Проект появится в `restored_project/`
6. Выполни: `cd restored_project && flutter pub get && npm install`

Пароль БД — в `backup_config.env` внутри архива (скопируется автоматически).
