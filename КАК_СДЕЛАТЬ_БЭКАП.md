# Как сделать полный бэкап Restodocks

## Первый раз: настройка (5 минут)

Перед первым бэкапом **один раз** сделай настройку — иначе база данных не попадёт в архив.

**Подробная инструкция:** открой **`НАСТРОЙКА_БЭКАПА_ОДИН_РАЗ.md`**

Кратко:
1. Установи **Postgres.app** — https://postgresapp.com (скачай → перетащи в Programs → Initialize)
2. Создай файл **backup_config.env** с паролем БД из Supabase Dashboard

---

## Бэкап (1 клик)

1. Двойной клик по **`Backup_Restodocks.command`**
2. Подожди 1–3 минуты
3. Готово. Появится `backup_YYYYMMDD_HHMMSS_COMPLETE.tar.gz`

В конце должно быть: **«База данных PostgreSQL ✓»**

---

## Как вернуть всё (полное восстановление)

Чтобы вернуться к сохранённому состоянию — **код, БД, конфиг**:

1. Открой терминал в папке проекта
2. Запусти один из вариантов:

**Из последнего архива:**
```bash
./restore_all.sh
```

**Из конкретного архива:**
```bash
./restore_all.sh backup_20260225_183212_COMPLETE.tar.gz
```

**Только откат кода** на checkpoint (без БД):
```bash
./restore_all.sh --checkpoint
```

Скрипт распакует архив, восстановит код, .env, миграции, спросит про БД (если есть дамп). Всё вернётся как было на момент бэкапа.
