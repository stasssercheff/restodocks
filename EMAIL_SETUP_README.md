# Настройка подтверждения email в Restodocks

## Обзор процесса регистрации

1. **Подтверждение email**: Пользователь получает письмо с ссылкой для подтверждения
2. **Приветственное письмо**: После подтверждения приходит второе письмо с PIN-кодом (для владельцев) или данными доступа (для сотрудников)

## Настройка в Supabase Dashboard

### 1. Включение подтверждения email

1. Перейдите в **Authentication → Providers → Email**
2. Включите опцию **"Confirm Email"**
3. Сохраните изменения

### 2. Настройка SMTP для отправки писем

1. Перейдите в **Authentication → SMTP Settings**
2. Выберите провайдера (Resend, SendGrid, AWS SES и т.д.)
3. Введите API ключ и настройки SMTP
4. Для **Resend**:
   - API Key: ваш ключ от Resend
   - Host: smtp.resend.com
   - Port: 587
   - Username: resend
   - Password: ваш API ключ

### 3. Переменные окружения

Добавьте в **Project Settings → Environment Variables**:
```
RESEND_API_KEY=ваш_resend_api_ключ
```

## Развертывание Edge Functions

```bash
# Из корневой папки проекта
cd restodocks_flutter/supabase

# Развернуть функции
supabase functions deploy send-email
supabase functions deploy send-welcome-email
```

## Запуск SQL миграции

1. Перейдите в **SQL Editor** в Supabase Dashboard
2. Выполните содержимое файла `setup_email_confirmations.sql`

## Тестирование

1. Зарегистрируйте нового пользователя через приложение
2. Проверьте получение письма с подтверждением
3. Перейдите по ссылке в письме
4. Проверьте получение приветственного письма

## Структура email шаблонов

### Подтверждение email (автоматически через Supabase)
- Тема: "Confirm your email"
- Содержит ссылку для подтверждения

### Приветственное письмо владельцу
- Тема: "Регистрация компании в системе Restodocks"
- Содержит: название компании, PIN-код, логин/пароль, инструкции

### Приветственное письмо сотруднику
- Тема: "Доступ к корпоративному пространству [Название компании]"
- Содержит: название компании, логин/пароль

## Устранение неполадок

### Письма не приходят
- Проверьте настройки SMTP
- Убедитесь, что API ключ верный
- Проверьте логи Edge Functions в Supabase Dashboard

### Webhook не срабатывает
- Проверьте, что функция развернута
- Проверьте логи в SQL Editor
- Убедитесь, что триггер создан

### PIN-код не отображается
- Проверьте, что у компании есть pin_code в таблице establishments
- Убедитесь, что сотрудник привязан к компании
