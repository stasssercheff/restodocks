# Парсер ТТК: цикл «исправили — снова не работает»

## Проблема

Файлы **Свекла**, **Цезарь**, **Сливочный крем** приходят в чат много раз. Каждый раз ответ «исправлено», но при повторной загрузке парсер снова работает неверно.

## Что уже есть в коде

| Карточка | Формат | Шаблон/тест | Файл-фикстура |
|----------|--------|-------------|---------------|
| Салат Цезарь | DOCX, ГОСТ 2-row | `tt_parse_templates` (наименование сырья \| расход на 1 порцию) | `test/fixtures/Техкарта_Салат_Цезарь.docx` |
| Печеная свекла | XLS, iiko/1С | `tt_parse_templates` (№\|наименование\|ед.изм.\|брутто) | только rows в тесте |
| Сливочный крем | PDF, Shama.Book | parseTtkByTemplate + извлечение названия | нет |
| Мясная к пенному | как печеная свекла | в seed | нет |

## Почему «снова не работает»

1. **Твои реальные файлы и фикстуры отличаются**  
   Тесты проходят на искусственных данных, а не на твоих DOCX/XLS/PDF.

2. **PDF ломает структуру**  
   Для PDF текст вытягивается в линейный поток, затем `pdfTextToRows` пытается восстановить таблицу. Для Shama.Book (2-row header, merged cells) это часто даёт неверные строки и колонки.

3. **Формат файла**  
   Цезарь и Свекла могут приходить как PDF, а шаблоны заточены под DOCX/XLS. PDF идёт другим путём и не использует те же шаблоны.

## Что нужно сделать

### Шаг 1: Файлы-фикстуры

Файлы могут лежать в `test/fixtures/` или в `test/`:

- `печенная свекла.xls`
- `Техкарта_Салат_Цезарь.docx` (уже есть)
- `ТК_на_Сливочный_крем_для_мафинов_и_дэкоров_п_ф,_№_.pdf`

Для XLS и PDF тесты вызывают Supabase Edge Functions — нужен доступ к Supabase.

### Шаг 2: Тесты по реальным файлам

После появления фикстур можно сделать тесты, которые:

- парсят эти конкретные файлы;
- проверяют, что dish name и количество ингредиентов совпадают с ожидаемыми.

### Шаг 3: Диагностика

Перед правками парсера добавить логи:

- какой `header_signature` получили;
- какие шаблоны найдены;
- почему отклонили строки/колонки.

Это даст возможность чинить парсер, опираясь на факты, а не на угадывание.

## Проверка пути: приложение vs тесты

Путь одинаковый:
- **Excel/XLS/DOCX**: `parseTechCardsFromExcel` → `_tryParseByStoredTemplates` (parse-ttk-by-templates) или `parseTtkByTemplate`
- **PDF**: `parseTechCardsFromPdf` → ai-parse-tech-cards-pdf

Если тесты проходят, а в приложении — нет, проверь:

1. **Edge Functions задеплоены?**
   ```bash
   npx supabase functions deploy parse-xls-bytes --project-ref osglfptwbuqqmqunttha
   npx supabase functions deploy parse-ttk-by-templates --project-ref osglfptwbuqqmqunttha
   npx supabase functions deploy ai-parse-tech-cards-pdf --project-ref osglfptwbuqqmqunttha
   ```

2. **В браузере** — DevTools → Console: ошибки при загрузке файла? (401, 500, CORS, timeout)

3. **Сообщение об ошибке** — при «Не удалось распознать» показывается `(reason)`. Например: `empty_text`, `invoke_null`, `extraction_failed` — по ним можно судить о причине.
