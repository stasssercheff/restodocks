# Безопасность: что делает владелец

Всё, что можно было сделать в коде и в репозитории, уже сделано и влито в **`main`** (включая `.github/dependabot.yml`, миграции RLS Phase 3, скрипты аудита). Дальше — только то, куда нужен **ваш вход** в сервисы.

---

## Сейчас: один заход в конце (15–25 минут)

Сделайте **по порядку**, когда будет время — **не обязательно** дробить на этапы.

1. **GitHub** → репозиторий Restodocks → **Settings** → **Code security and analysis**  
   Включите: **Dependency graph**, **Dependabot alerts** (и по желанию **Dependabot security updates**).  
   Так начнут работать предупреждения по уязвимым пакетам; файл `.github/dependabot.yml` на `main` начнёт давать **еженедельные PR** с обновлениями зависимостей.

2. **Supabase** → ваш проект → **SQL Editor**  
   Если Phase 3 ещё **не** выполняли на этой базе: откройте в репозитории файл  
   `supabase/migrations/20260430100000_security_hardening_phase3_pos_stock_errors_rls.sql`,  
   скопируйте **весь** текст в SQL Editor и выполните **одним запросом**.  
   Если этот SQL вы уже запускали — **повторно не нужно**.

3. **По желанию в тот же день:** включите **MFA** на аккаунте **GitHub** и в **Supabase** (настройки профиля / безопасности).

4. **Cloudflare** (если пользуетесь): убедитесь, что Beta смотрит на ветку **`staging`**, Prod на **`main`** (как у вас в `docs/DEPLOY_RULES.md`).

На этом **текущий обозначенный процесс** по настройке безопасности из репозитория считается завершённым с вашей стороны.
