# Инструкция: подготовка к интеграции Restodocks с iiko

Цель: за время демо iiko (1 месяц) собрать всё необходимое для последующей интеграции — без спешки и «на глаз».

---

## 1. Что нужно выгрузить и сохранить

### 1.1. Номенклатура (товары/продукты)

- [ ] **Экспорт справочника товаров** — в любом доступном формате (Excel, CSV, XML)
- [ ] **Экспорт цен** — если хранятся отдельно
- [ ] Сохранить файлы в `docs/samples/iiko/` с понятными именами, например:
  - `iiko_nomenclature_YYYYMMDD.xlsx`
  - `iiko_prices_YYYYMMDD.csv`

**Что проверить в выгрузке:**
- Какие поля есть: код/артикул, название, единица измерения, группа, цена
- Как заданы единицы: кг, г, шт, л, мл
- Есть ли `grams_per_piece` или аналог для штучных товаров
- Есть ли деление на продукты и полуфабрикаты

### 1.2. Технологические карты (ТТК)

- [ ] **Экспорт ТТК** — минимум 5–10 карточек в разных форматах, если возможно
- [ ] Карточки должны включать:
  - Только продукты (без ПФ)
  - Продукты + ПФ (вложенные ТТК)
  - Блюда и полуфабрикаты
- [ ] Сохранить в `docs/samples/iiko/`:
  - `iiko_ttk_dish_1.xls`
  - `iiko_ttk_pf_1.xls`
  - и т.д.

**Что проверить:**
- Структура таблицы: колонки (брутто, нетто, выход, единицы)
- Как обозначены ПФ в составе: «п/ф», «полуфабрикат», отдельная колонка
- Есть ли блок «Технология приготовления»
- Есть ли связь ингредиента с кодом продукта из номенклатуры (ID, артикул)

### 1.3. Группы и категории

- [ ] **Справочник групп товаров** — как организована иерархия
- [ ] **Справочник групп блюд/ТТК** — как в iiko делятся блюда и ПФ
- Сохранить структуру (скрин или экспорт)

## 4. Краевые случаи

Проверить в iiko и зафиксировать:

## 7. Что это даст для интеграции

После выполнения инструкции:

1. **Импорт номенклатуры iiko** — известны формат и поля
2. **Импорт ТТК из iiko** — есть готовые образцы для парсера
3. **Маппинг** — чёткое соответствие iiko ↔ Restodocks
4. **Тесты** — samples/ для regression-проверок
5. **API** — при наличии документации и примеров можно проектировать синхронизацию
