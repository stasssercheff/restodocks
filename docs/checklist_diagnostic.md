# Диагностика «чеклист не сохраняется»

## Шаг 1: Проверить, сохраняются ли данные в БД

1. Создайте чеклист, введите название "Тест123", добавьте пункт "Пункт А", нажмите Сохранить.
2. Запомните ID чеклиста из URL (например `/checklists/97421b61-f927-41ff-8df6-458c27834901`).
3. Откройте **Supabase Dashboard** → **Table Editor** → `checklists`.
4. Найдите строку с этим ID. Обновилось ли поле `name` на "Тест123"?
5. Откройте таблицу `checklist_items`. Есть ли строки с этим `checklist_id`?

**Если ДА** — данные сохраняются, проблема в отображении (кэш списка).
**Если НЕТ** — данные не попадают в БД, проблема в сохранении.

## Шаг 2: Один ли проект Supabase

- В логе приложения: `url=https://osglfptwbuqqmqunttha.supabase.co`
- В Supabase Dashboard проверьте URL — тот же ли проект?
- Если проекты разные — приложение пишет в один проект, SQL применяется к другому.

## Шаг 3: RPC save_checklist

Убедитесь, что в Supabase выполнён `docs/apply_save_checklist_no_check.sql`.
В **SQL Editor** проверьте наличие функции:
```sql
SELECT proname FROM pg_proc WHERE proname = 'save_checklist';
```
Должна вернуться одна строка.
