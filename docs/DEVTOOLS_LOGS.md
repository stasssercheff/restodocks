# DevTools — логи и ошибки в консоли

## Почему много сообщений

- **devLog** и **debugPrint** — выводятся только в **Debug** (разработка). В Release/Production их нет.
- **Библиотеки** (Supabase, Flutter, и т.п.) могут писать свои логи.
- **Предупреждения** (жёлтые) часто безвредны — устаревшие API, лишние rebuild'ы и т.д.

## Что важно

| Тип | Опасность |
|-----|-----------|
| Красные ошибки с `Exception` / `Error` | Разбирать, если повторяются и влияют на работу |
| `401`, `403`, `500` в Network | Проблемы авторизации или сервера |
| CORS / blocked | Часто из-за настроек Supabase Auth / домена |
| Жёлтые предупреждения | В основном косметика, можно оставлять |
| Синие логи (devLog, debugPrint) | Только для отладки, в продакшене нет |

## Если что-то падает

1. Скопировать текст ошибки и стек.
2. Проверить, воспроизводится ли при каждом действии или только иногда.
3. Фильтровать по `Error`, `Exception`, `failed` — смотреть на реальные сбои.

## Уменьшение шума

- **Локально:** `flutter run --release` — почти без логов приложения.
- **Продакшен (Cloudflare):** билд в release, там нет `devLog` и `debugPrint`.

## План аудита логов (по желанию)

- Удалить лишние `debugPrint`/`devLog`.
- Оборачивать временные логи в `kDebugMode`.
- Добавить уровень (error/warn/info) и возможность отключать инфо-логи.
