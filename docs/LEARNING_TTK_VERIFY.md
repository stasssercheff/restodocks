# Обучение парсера ТТК — проверка работы

## Как работает обучение

1. **tt_parse_templates** — шаблоны колонок (№, продукт, брутто, нетто…). Сохраняются при первом успешном парсинге по ключевым словам или AI.
2. **tt_parse_learned_dish_name** — выученные позиции (где название блюда, product_col, gross_col, net_col). Сохраняются при правках в импорте.
3. **tt_parse_corrections** — правки original → corrected (названия блюд). Применяются при следующем парсинге того же формата.

## Что исправляет код, а что — обучение

**Только код (без обучения):**

- **unpdf:** замена pdf-parse на unpdf, `mergePages: false` и склейка страниц вручную — сохраняются переносы строк в PDF, таблица не склеивается в одну строку.
- **Граница таблицы и технология:** строки с «Выход на 1 порцию», «Технологический процесс» считаются концом таблицы; текст технологии берётся только из строк **после** этой границы (и из строк «перед» границей, если они выглядят как инструкция: «сахар пробить…», «добавить…»).
- **Выученная колонка технологии** в сохранённом шаблоне применяется только к строкам **после границы таблицы**, а не ко всем строкам (иначе в технологию попадали бы «1», «2», названия продуктов).

**Обучение (правки пользователя):**

- При **«Создать все»** на экране проверки импорта в БД сохраняются выбранные колонки (название блюда, продукт, брутто, нетто, **технология**). При следующем импорте **того же формата** (та же подпись заголовка):
  - если сработал встроенный шаблон — технология **дополнительно** подставляется из выученной колонки (из строк после границы таблицы);
  - если шаблон не сработал — парсинг идёт по сохранённому шаблону с выученными колонками.
- В SnackBar после «Создать все» при успешном сохранении обучения показывается: **«Обучение сохранено: продукт, брутто, нетто, технология — учтено для следующего импорта этого формата.»** Это и есть подтверждение, что обучение сработало.

## Почему обучение «почти ни разу не срабатывало» на практике

**Причина:** При выборе результата парсинга (Excel/таблицы) всегда побеждал **встроенный шаблон**, если он давал карточки с ингредиентами. Выученные колонки (`tt_parse_learned_dish_name`) использовались только когда встроенный шаблон ничего не находил — тогда вызывался EF `parse-ttk-by-templates`, который и подставляет обучение. При повторном импорте того же файла встроенный шаблон снова находил «что-то» (неправильные колонки, но с ингредиентами), и этот результат брали — обучение не применялось.

**Исправление:** Приоритет у результата **обучения**: если для подписи заголовка есть сохранённые колонки и EF вернул карточки с ингредиентами (`listByStored`), используем их; иначе — как раньше (встроенный шаблон или fallback). После правок и «Создать все» при повторном импорте того же формата данные должны вставать в нужные колонки.

(Для применения обучения через EF нужна ещё запись в `tt_parse_templates` — она создаётся при первом успешном парсинге формата по ключевым словам или после AI.)

## Обучение «в принципе» — все форматы

**Когда обучение сохраняется:**

- **«Создать все»** на экране проверки импорта (PDF, Excel, текст) — колонки и правки пишутся в БД.
- **Сохранение карточки из редактора**, если карточка открыта из импорта (`initialFromAi` + `initialHeaderSignature`) — вызывается `learnDishNamePosition`, данные попадают в `tt_parse_learned_dish_name`. В тосте показывается: «Карточка создана. Обучение сохранено: … — учтено для следующего импорта этого формата.»

**Когда обучение применяется:**

- **PDF:** после разбора по шаблону подставляется технология из выученной колонки (`getTechnologyFromRowsUsingColumn` в EF). При следующей загрузке того же PDF технология уже подтягивается.
- **Excel / таблицы:** после получения карточек (встроенный шаблон или сохранённый) вызывается `_enrichTechnologyFromLearned`: читается `tt_parse_learned_dish_name` по подписи заголовка, из строк после границы таблицы берётся текст из выученной колонки технологии и подставляется в первую карточку. Если парсинг идёт через EF `parse-ttk-by-templates` (когда встроенный шаблон не сработал), выученные колонки используются уже внутри `tryParseByStoredTemplates`.
- **Правки названий** (`tt_parse_corrections`) применяются при любом формате в `_applyParseCorrections`.

Итого: обучение единое для всех источников (PDF, Excel, текст); сохраняется при «Создать все» и при сохранении из редактора; применяется при следующем импорте того же формата (подпись заголовка).

## Парсинг и обучение

Логика парсинга в `parse_ttk_template.ts`, `try_stored_ttk_templates.ts`, `ai-parse-tech-cards-pdf` использует и встроенные правила (границы, колонки по ключевым словам), и данные из `tt_parse_learned_dish_name` (в т.ч. колонка технологии при уже сработавшем шаблоне). На клиенте для Excel/таблиц обогащение технологии из выученной колонки выполняет `_enrichTechnologyFromLearned`.
